NSUserDefaults standardUserDefaults的使用

发表于 2016-03-18 | 分类于 Home ， iOS 
NSUserDefaults用于存储数据量小的数据，例如用户配置。并不是所有的东西都能往里放的，只支持：NSString,NSNumber, NSDate, NSArray, NSDictionary (之前有面试官问我这个问题来着：“用户偏好设置什么时候用？怎么用？可以存放NSMutableArray/Dictionary吗？我没有回答上来”)，详细方法可以查看类文件。

NSUserDefaultsstandardUserDefaults用来记录一下永久保留的数据非常方便，不需要读写文件，而是保留到一个NSDictionary字典里，由系统保存到文件里，系统会保存到该应用下的/Library/Preferences/gongcheng.plist文件中。需要注意的是如果程序意外退出，NSUserDefaultsstandardUserDefaults数据不会被系统写入到该文件，不过可以使用［[NSUserDefaultsstandardUserDefaults] synchronize］命令直接同步到文件里，来避免数据的丢失。

[code lang=”objc”] 一、将数据存储到NSUserDefaults： //UISwitch - (IBAction)switchChanged:(id)sender{ NSUserDefaults *userDefaults = [NSUserDefaults standardUserDefaults]; [userDefaults setBool:_theSwitch.on forKey:@”switchValue”]; } //UITextField - (IBAction)inputChanged:(id)sender{ NSUserDefaults *userDefaults = [NSUserDefaults standardUserDefaults]; [userDefaults setObject:_textField.text forKey:@”inputValue”]; } [/code]

二、读取NSUserDefaults中的数据：

[code lang=”objc”] UISwitchNSUserDefaults *userDefaults = [NSUserDefaults standardUserDefaults]; BOOL sw = [userDefaults boolForKey:@”switchValue”]; [_theSwitch setOn:sw]; UITextFieldNSString *str = [userDefaults stringForKey:@”inputValue”]; [_textField setText:str]; [/code]

registerDefaults:方法是注册偏好设置的子集，它是不写入到plist文件中的，但在ND中取确实能取到。
也就是说plist文件中看到的数据是你显示的设置进去的。

取消TableView Section的headerView和footerView的滞留效果

发表于 2016-03-17 | 分类于 Home ， iOS

TableView Section的headerView和footerView的粘性效果是iOS本身的特性，有的时候这种效果很好，但是有时候我们却不需要这种效果。

[code lang=”objc”] -(void)scrollViewDidScroll:(UIScrollView )scrollView { if (scrollView == self.tableView) { UITableView *tableview = (UITableView \)scrollView; CGFloat sectionHeaderHeight = 64; CGFloat sectionFooterHeight = 120; CGFloat offsetY = tableview.contentOffset.y; if (offsetY >= 0 && offsetY <= sectionHeaderHeight) { tableview.contentInset = UIEdgeInsetsMake(-offsetY, 0, -sectionFooterHeight, 0); }else if (offsetY >= sectionHeaderHeight && offsetY <= tableview.contentSize.height - tableview.frame.size.height - sectionFooterHeight) { tableview.contentInset = UIEdgeInsetsMake(-sectionHeaderHeight, 0, -sectionFooterHeight, 0); }else if (offsetY >= tableview.contentSize.height - tableview.frame.size.height - sectionFooterHeight && offsetY <= tableview.contentSize.height - tableview.frame.size.height) { tableview.contentInset = UIEdgeInsetsMake(-offsetY, 0, -(tableview.contentSize.height - tableview.frame.size.height - sectionFooterHeight), 0); } } } [/code]

最近使用TableView注意点总结

发表于 2016-03-17 | 分类于 Home ， iOS

最近与TableView接触比较多，同时也发现很多容易出BUG的地方，放在这里总结一下。

1. [tableView dequeueReusableCellWithIdentifier:CELL_REUSEID_MRLIST]方法的使用

这个方法是从已经创建过的cell中选出合适的cell用来复用，也就是所谓的tableview的复用机制，也就是说这个方法在

(UITableViewCell *) tableView:(UITableView *)tableView cellForRowAtIndexPath:
(NSIndexPath *)indexPath 调用的时候是不会执行的。

2.

[[MrListCell alloc] initWithStyle:UITableViewCellStyleDefault reuseIdentifier:CELL_REUSEID_MRLIST]方法的使用

这个方法是用来创建新的cell的，一般来说，一个tableview加载的时候需要显示5个cell那么这个方法就会创建5个新的cell，需要注意的是在这个方法里面，也就是 if （cell == ni） {  }这个方法里面调用的方法和使用的内容要注意，如果在这里面写的有创建控件或者分配空间的方法就会出现cell的内容重叠或者空间浪费的情况。
总结一下就是一定要分清楚tableview创建的时候用到的这两个方法，然后在书写代码的时候一定要把创建和赋值给区分开，避免造成空间浪费的情况，对了，还有一点就是，不到万不得已的情况下不要关闭tableview的重用机制。
