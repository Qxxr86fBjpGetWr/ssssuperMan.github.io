OpenAI的Prompt Engineering 课程

5. 模型推理

大模型有将一个文本作为输入，然后进行提取标签、提取名字、理解文本情感等能力。比如想要提取一段文本中的态度是正面/负面的操作，在传统的机器学习工作流程中，需要先收集标签数据，然后训练模型、部署模型，这是一个复杂的过程。另外分析情感、提取标签都需要分别训练和部署一个单独的模型。对于大语言模型来说都可以处理，这给开发应用程序带来了巨大的提速。

下面我们来看一个例子，我们根据一段用户的评论，来提取评论的态度是否是积极的，用户是否表达了愤怒，产品的名称、产品的制造商等信息。

我们看看大模型是怎么输出的：

模型按照要求出色的完成了任务，然后 json 结构化的形式输出，我们可以拿到结果很轻松的在 python 或者其他语言框架中作为输入处理结果。

接下来再看一个例子，我们输入一段虚构的新闻报道，让大模型来分析这段文字的主题，并且罗列这个文章五个相关的主题列表。

我理解这就是继续迭代提示词的动机，我们可以继续完善提示词，通过提供案例，约束输出来实现不同模型间输出的差异。吴恩达老师在这里提到了一个概念，我们上面的这种工程，没有提供任何有标签的训练数据，这也被称之为零示例工程。

在短短几分钟之内，建立了多个能力，对文本进行推断，要想实现这些能力，在之前需要一名熟练的机器学习开发人员数天或者数周的时间才能完成。所以，无论对于熟练的机器学习开发人员还是机器学习新手而言，这都是一个令人兴奋的消息。现在可以使用提示词非常快速的对复杂的自然语言处理任务做出推断处理。
