iOS开发：使用XMPP登录时支持IPV6处理

Posted on 2016-06-06 Edited on 2019-10-21 In iOS | 阅读量: 次

五月份Apple就发布了声明，六月一号之后所有APP需要支持IPV6，之前并没有留意到这个通知，直接导致上周APP提交之后被拒了。
Apple拒绝邮件里面是这样说的：

2.2 - Apps that exhibit bugs will be rejected
2.2 Details

We discovered one or more bugs in your app when reviewed on iPhone running iOS 9.3.2 on both Wi-Fi and cellular networks.

- The loading indicator spins indefinitely when users tapped on the login button.

Next Steps

Please run your app on a device to identify the issue(s), then revise and resubmit your app for review.

Apps are reviewed on an IPv6 network. Please ensure that your app supports IPv6 networks, as IPv6 compatibility is required.

For additional information about supporting IPv6 Networks, please refer to Supporting iPv6 DNS64/NAT64 Networks.

For a networking overview, see About Networking.

大概就是：

1.“点击登录按钮的之后没有反应，HUB提示控件一直转圈”。

2.你的APP得支持IPV6.

我一开始感觉是XMPP的问题，因为XMPP的一系列的代理方法都没有执行，为了验证先配置了IPV6的环境（参考：https://developer.apple.com/library/mac/documentation/NetworkingInternetWeb/Conceptual/NetworkingOverview/UnderstandingandPreparingfortheIPv6Transition/UnderstandingandPreparingfortheIPv6Transition.html#//apple_ref/doc/uid/TP40010220-CH213-SW1）

果然在IPV4环境下功能正常连接上IPV6上之后就出现了不能登录的问题，具体表现就是第一条所述。

我尝试了一下的方法：

1.更新XMPP。因为当初项目并没有使用CocoaPods所以更新起来非常麻烦，无果。

2.把IPV4地址转换成IPV6。无果。

最终的解决办法是：

1.网络请求的宏，使用指向IP地址的域名，而不是IP地址。

2.因为XMPP里面是用Socket做的链接所以还要在XMPPStream文件的这个方法稍作修改： [code lang=”objc”] - (id)init { if ((self = [super init])) { // Common initialization [self commonInit]; // Initialize socket asyncSocket = [[GCDAsyncSocket alloc] initWithDelegate:self delegateQueue:xmppQueue]; [asyncSocket setIPv4PreferredOverIPv6:NO];} return self; } [/code]

最终测试完美解决！
